Settl → shadcn/ui

How to rebuild the Settl prototype on the shadcn/ui stack (Next.js + Tailwind + Radix). The prototype's design tokens and components map nearly 1:1 — this doc gives the theme CSS, the component mapping, and the behavioral details that are easy to miss.

1 · Stack & fonts

  • shadcn/ui default preset, CSS variables mode (cssVariables: true)
  • Fonts via next/font: Bricolage Grotesque (UI, headings) + Spline Sans Mono (all amounts — map to font-mono)
  • Radius: the design is rounder than shadcn default — set --radius: 1rem
  • Locale: sv-SE throughout; amounts via toLocaleString('sv-SE') + non-breaking space before ”kr”

2 · Theme tokens (globals.css)

Mint is the default direction. Paper and Citrus ship as opt-in theme classes. shadcn has no ”success” token — Settl needs one for ”owed to you” amounts, so --success is added as a custom token.

4 · Behavior that must survive the port

  • Per-pair settlement. An entry is not a single ”settled” boolean. Each entry holds debts (from → to). Settling up with one person closes only the debts between that pair; a 3-way expense stays open for the third person. The entry counts as settled when all its debts are closed.
  • Recurring = template + posts. A recurring cost is a template (amount, cadence, split, payer, next date). Each cycle it posts an ordinary ledger entry and re-splits. Editing the split affects future cycles only. Pausing stops posting without deleting history.
  • Pure ledger. No payments in-app. ”Markera som reglerad” is the only settlement action; copy references Swish/kontanter/banköverföring.
  • Event-triggered nudges only. Three triggers: recurring posts within 5 days; a new expense ≥ 1 500 kr; a pair balance crossing 750 kr. Two copy tones (gentle/direct) exist as a setting. Never scheduled daily reminders.
  • Open trust model. Anyone in the household adds entries directly — no approval flow anywhere.
  • Split validation. Percent must sum to 100 (±0.5), exact amounts must sum to the total (±0.05 kr); the running ”fördelat” hint flips between success/destructive color.

5 · Suggested data model

type Household = { id: string; name: string; memberIds: string[] }

type Split =
  | { mode: 'equal' }
  | { mode: 'percent' | 'amount'; vals: Record<MemberId, number> }

type Entry = {
  id: string; householdId: string
  type: 'expense' | 'iou' | 'recurring-post'
  title: string; amount: number; date: string
  paidBy?: MemberId              // expense / recurring-post
  from?: MemberId; to?: MemberId // iou
  split?: Split
  recurringId?: string           // link to template
  settledPairs: PairKey[]        // per-pair settlement, not a boolean
}

type Recurring = {
  id: string; householdId: string
  title: string; amount: number
  cadence: 'monthly' | 'biweekly' | 'weekly'
  nextPostDate: string; paidBy: MemberId
  split: Split; active: boolean
}
